Level 1 / Project 09 — JSON Settings Loader Load application settings from a JSON string, merge with defaults, and validate required keys. Browser adaptation: Instead of reading JSON from a file, you will parse a JSON string using json.loads(). The same merge and validation logic applies. 1. parse_json(text) - Parse a JSON string using json.loads() - If the JSON is invalid, raise ValueError with a helpful message - Return the parsed dictionary 2. merge_settings(defaults, overrides) - Copy defaults, then apply overrides on top - Do NOT modify the original defaults dict - Return the merged dictionary 3. validate_settings(settings, required) - Check that all keys in the "required" list exist in settings - Return a list of missing key names (empty list if all present) 4. settings_diff(defaults, settings) - Return a list of strings showing what changed from defaults - Format: " key: old_value -> new_value" for changed keys - Format: " key: (new) value" for keys not in defaults Defaults to use: {"app_name": "MyApp", "debug": False, "log_level": "INFO", "max_retries": 3, "timeout_seconds": 30, "port": 8080} Required keys: ["app_name", "port"] Examples: merge_settings({"a": 1, "b": 2}, {"b": 99, "c": 3}) -> {"a": 1, "b": 99, "c": 3} validate_settings({"app_name": "X"}, ["app_name", "port"]) -> ["port"]
