Level 1 / Project 01 — Input Validator Lab Write three validator functions that check common input formats using string methods. 1. validate_email(email) - Must contain exactly one "@" - Must have text before and after the "@" - The part after "@" must contain at least one "." - No spaces allowed - Return a dict: {"value": email, "type": "email", "valid": True/False, "errors": [...]} 2. validate_phone(phone) - Extract only digits from the string - Must have exactly 10 digits - Return a dict: {"value": phone, "type": "phone", "valid": True/False, "errors": [...]} 3. validate_input(line) - Parse a line like "email: user@example.com" (split on first ":") - Dispatch to the correct validator based on type - Return {"raw": line, "error": "..."} if format is wrong or type is unknown Examples: validate_email("user@example.com") -> {"valid": True, ...} validate_email("bad-email") -> {"valid": False, "errors": ["must contain exactly one @"]} validate_phone("555-123-4567") -> {"valid": True, ...} validate_input("email: test@t.com") -> {"type": "email", "valid": True, ...} Browser note: The real project reads from a file.
