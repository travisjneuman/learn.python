# 20 - Curriculum Changelog
Home: [README](../README.md)

This file tracks curriculum evolution and release decisions.

## Versioning strategy
- Major: structural or assessment model changes.
- Minor: content additions and improvements.
- Patch: typo, link, formatting, and non-structural fixes.

## Quarterly release process
1. Gather manual progress notes and feedback.
2. Prioritize high-impact fixes.
3. Run quality gates:
  - markdown links,
  - smoke checks,
  - readability audit,
  - rubric consistency check.
4. Publish release notes.

## Ownership
- Final approval owner: (curriculum maintainer).
- Proposed changes should include rationale, impact, and rollback plan.
